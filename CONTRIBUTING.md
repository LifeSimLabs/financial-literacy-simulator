# Contributing to Financial Literacy Simulator

First off, thank you for considering contributing to the Financial Literacy Simulator! It's people like you that make open source such a great community.

## Git Workflow
1. **Fork** the repo on GitHub.
2. **Clone** the project to your own machine.
3. **Commit** changes to your own branch.
4. **Push** your work back up to your fork.
5. Submit a **Pull Request** so that we can review your changes.

## Branch Naming
We follow a structured branch naming convention:
- `feature/<feature-name>` for new features
- `bugfix/<bug-name>` for bug fixes
- `docs/<doc-name>` for documentation updates
- `research/<topic>` for research and exploration

## Commit Conventions
We use [Conventional Commits](https://www.conventionalcommits.org/). This leads to more readable messages that are easy to follow when looking through the project history.
Format: `<type>[optional scope]: <description>`
Examples:
- `feat: add compound interest calculation`
- `fix: resolve issue with inflation adjustment`
- `docs: update architecture diagram`

## Issue Creation
Before creating a new issue, please check existing issues to avoid duplicates. We have templates for:
- Bug Reports
- Feature Requests
- Research Proposals
- Design Proposals

## Discussions
If you have an idea, a question, or want to discuss architecture, please use GitHub Discussions. We have categories for Ideas, Research, Architecture, and Questions.

## Pull Requests
- Ensure your code adheres to our coding standards.
- Update documentation if you are changing functionality.
- Fill out the provided Pull Request template completely.\n
